# 一体化架构

我们团队最核心的主线。**BI 和 AI 的负载，为什么应该跑在同一个湖仓上**，这节给答案和案例。

## 待补

- Lake + Vector 融合架构（向量索引下沉到表格式）
- 统一 Catalog 策略（同一套注册中心同时管关系表与向量表）
- 多模数据建模（文本、图像、音频、视频 + 结构化字段）
- 计算统一（同一引擎读关系表与向量表）
- 业内案例拆解：Databricks Mosaic、Snowflake Cortex、OpenAI 私有湖仓
